Angabe: Infrastruktur-Monitoring-Programm<br>

Erstelle eine Klassendiagramm zu einem Infrastruktur-Monitoring-Programm<br>

Komponenten/Funktionalität sollte geplant werden:<br>
	1. Das zu prüfende Item (zB. Server-IP, Web-URL, etc.)<br>
	2 .Kategorie (zB. Client, Server, Webservice, Dienste, etc.)<br>
	3. Prüfart (zB. Ping, HTTP-GET, WMI, etc.)<br>
	4. Zeitplan, Intervall<br>
	5. Notification (zB. Mail, SMS, Telegram, etc.)<br>
	6. Eskalationssystem (Regeln einfach selbst erfinden) Kann sich bspw. an der Kategorie orientieren, am Schweregrad, Zeit, etc.<br>
	7. Rechtesystem für Benutzer (optional)<br>
	8. Implementiere (rudimentär) das Klassendiagramm (>= 5 Klassen) mit einigen beispielhaft(!) implementierten Funktionen<br>
	9. Mindestens drei sinnvolle unterschiedliche Unit-Tests erstellen für die beispielhaft implementierten Funktionen<br>
	10. Präsentation (~10 Minuten)<br>
	11. Ausreichende Dokumentation als Markdown-Datei (Beschreibung der Applikation, Mitwirkende, Klassendiagramm<br>
	12. **Abgabe am eCampus: EINE Person pro Gruppe!**<br>
